#Tomcat
* <b>동적인 데이터를 처리하는 웹서버</b>
* WAS(Web Application Server)이며, 웹서버와 웹 컨테이너의 결합으로 다양한 기능을 컨테이너에 구현하여 다양한 역할을 수행할 수 있는 서버
* 자바 서블릿 실행 -> JSP코드가 포함되어 있는 웹 페이지 생성
* 웹 서버에서 넘어온 동적인 페이지를 읽어 프로그램 실행 -> 그 결과를 html로 재구성, 아파치에게 반환


## 아파치 == 웹서버 톰캣 == WAS(Web Application Server)

* 톰캣만을 사용하면 비용이 큼. (html, jsp 파일을 같은 것으로 보고 처리하기 때문에 대량의 트래픽에선 많은 비용이 든다.)

* 아파치로 정적인 페이지를 수행하고, 톰캣으로 동적인 페이지를 수행시키는 것이 적합

아파치와 톰캣을 연동한다는 것은 동일한 포트로 운영한다는 의미로, 보통 이미지나 HTML 요소는 아파치에서 처리가 되고, 사용자의 데이터 처리와 같은 일은 톰캣에서 처리하게 한다



WAS(Web Application Server)란?
웹 애플리케이션 서버는 웹 애플리케이션과 서버 환경을 만들어 동작시키는 기능을 제공하는 소프트웨어 프레임워크이다. 인터넷 상에서 HTTP를 통해 사용자 컴퓨터나 장치에 애플리케이션을 수행해 주는 미들웨어
* 웹 서버 + 웹 컨테이너

## 웹 서버와 WAS의 차이점
 - 웹 서버와 WAS를 구별 짓는 것은 동적 서버 콘텐츠를 수행하는가? 만약 수행한다면 WAS
 - 웹 서버 : 정적인 HTML이나 이미지를 제공하는 서버.
   WAS : 동적인 처리를 담당하는 서버.
