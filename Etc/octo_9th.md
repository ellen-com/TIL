# Maven과 Gradle의 차이
일단, Maven과 Gradle 모두 자바 빌드 도구입니다.

## Maven
프로젝트에 필요한 모든 Dependency(종속성)를 리스트의 형태로 관리합니다.
쉽게 말해서 프로젝트에서 사용할 수 많은 라이브러리들을 관리해주는 도구입니다. 
* pom.xml 파일에 필요한 jar나 class path를 선언만 해주면 Maven이 자동으로 가져와서 우리의 프로젝트에서 사용할 수 있습니다.
* 라이브러리들과 연관된 라이브러리들까지 모두 다 연동이 되어서 관리됩니다.
* xml 파일은 txt파일이기 때문에 오타가 났을 경우 오류를 찾기 어렵습니다.

## Gradle
Gradle은 groovy를 이용한 빌드 자동화 시스템입니다.
빌드 스크립트가 프로그래밍 언어와 비슷해서 개발자에겐 좀 더 친숙한 빌드도구입니다.


Build라는 동적인 요소를 XML로 정의하기에는 어려운 부분이 많다.
* 설정 내용이 길어지고 가독성 떨어짐
* 의존관계가 복잡한 프로젝트 설정하기에 부적절
* 상속구조를 이용한 멀티 모듈 구현
* 특정 설정을 소수의 모듈에서 공유하기 위해서는 부모 프로젝트를 생성하여 상속하게 해야 함 (상속의 단점 생김)

Gradle은 Groovy를 사용하기 때문에, 동적인 빌드는 Groovy 스크립트로 플러그인을 호출하거나 직접 코드를 짜면 된다.
* Configuration Injection 방식을 사용해서 공통 모듈을 상속해서 사용하는 단점을 커버했다.
* 설정 주입 시 프로젝트의 조건을 체크할 수 있어서 프로젝트별로 주입되는 설정을 다르게 할 수 있다.
